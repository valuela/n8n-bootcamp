# The Complete n8n Bootcamp

## Curriculum Roadmap

**Version:** 1.0

---

# 1. Curriculum Purpose

This document defines the complete learning journey of **The Complete n8n Bootcamp**.

Unlike the Bootcamp Specification, which defines educational standards, or the Competency Matrix, which defines employer expectations, this curriculum defines **how students progress from beginner to job-ready Automation Engineer**.

Every module exists for a specific purpose and contributes measurable competencies that prepare students for real-world automation engineering roles.

The curriculum is intentionally sequenced so that every concept builds upon previous knowledge while reinforcing practical engineering skills through projects, labs, assessments, and portfolio development.

---

# 2. Learning Journey

Every student progresses through five stages.

```text
BEGINNER
↓

BUILDER
↓

ENGINEER

↓

PROFESSIONAL

↓

JOB READY
```

Each stage represents a shift in mindset, responsibility, and capability.

---

## Stage 1 — Beginner

### Objective

Learn how automation works.

### Focus

- Automation concepts
- n8n fundamentals
- JSON
- Data flow
- Basic workflow execution

### Student Mindset

"I understand how automation systems work."

---

## Stage 2 — Builder

### Objective

Build working automations confidently.

### Focus

- Core nodes
- Expressions
- Conditions
- Loops
- Data transformation
- APIs
- Webhooks

### Student Mindset

"I can build reliable workflows independently."

---

## Stage 3 — Engineer

### Objective

Design automation systems.

### Focus

- Architecture
- Databases
- AI
- Error handling
- Reusability
- Scalability

### Student Mindset

"I can design solutions rather than just workflows."

---

## Stage 4 — Professional

### Objective

Think like an Automation Engineer.

### Focus

- Business analysis
- Documentation
- Deployment
- Monitoring
- Communication
- Portfolio development

### Student Mindset

"I understand how automation is delivered in professional environments."

---

## Stage 5 — Job Ready

### Objective

Demonstrate professional competency.

### Focus

- Capstone
- Mock interviews
- Technical presentations
- Architecture defense
- End-to-end project delivery

### Student Mindset

"I am prepared to contribute professionally as an Automation Engineer."

---

# 3. Curriculum Structure

The curriculum consists of thirteen progressive modules.

Each module contains:

- Learning outcomes
- Lessons
- Guided labs
- Independent labs
- Design challenges
- Portfolio projects
- Assessments
- Interview preparation
- Reflection

Students should complete modules sequentially.

Every module includes a consistent set of learning activities.

| Activity            | Purpose                                                   |
| ------------------- | --------------------------------------------------------- |
| Lessons             | Introduce new concepts                                    |
| Guided Labs         | Build confidence with instructor support                  |
| Independent Labs    | Reinforce concepts independently                          |
| Design Challenges   | Solve business problems without step-by-step instructions |
| Portfolio Projects  | Produce employer-ready work                               |
| Debugging Exercises | Develop troubleshooting skills                            |
| Reflections         | Strengthen engineering judgment                           |
| Interview Practice  | Build communication and confidence                        |

---

# 6. Assessment Strategy

Students are assessed through practical evidence rather than memorization.

Assessment methods include:

- Guided labs
- Independent labs
- Workflow reviews
- Design challenges
- Debugging exercises
- Documentation reviews
- Portfolio projects
- Mock interviews
- Capstone presentation

Practical application takes priority over theoretical recall.

---

# 7. Curriculum Completion Standard

Students are considered graduates when they can independently:

- Analyze business requirements.
- Design automation architecture.
- Build production-quality workflows.
- Integrate APIs, databases, and AI.
- Handle errors and operational concerns.
- Document and present technical work professionally.
- Defend architectural decisions.
- Deliver a complete business automation solution.

Completion is measured by demonstrated professional capability rather than the number of completed lessons.

# 8. Continuous Improvement

This curriculum is intended to evolve with industry practices.

Future revisions should preserve the learning philosophy while incorporating improvements based on:

- Employer feedback
- Graduate outcomes

- AI advancements
- Community contributions
- Industry trends

All changes should be documented in `CHANGELOG.md`.
