# PROGRAM_OUTCOMES.md

# The Complete n8n Bootcamp

## Program Outcomes and Job-Readiness Framework

**Version:** 1.0

---

# 1. Purpose

This document defines the measurable professional outcomes of The Complete n8n Bootcamp.

It explains what graduates should be able to do, how those abilities are demonstrated, and how each module contributes to job readiness.

---

# 2. Graduate Capabilities

By the end of the bootcamp, graduates should be able to:

- Analyze a business process and identify automation opportunities.
- Translate requirements into workflow architecture.
- Build reliable n8n workflows using triggers, branching, loops, and reusable components.
- Transform, validate, and map structured data.
- Integrate REST APIs using authentication, pagination, and error handling.
- Build webhook-driven and scheduled automations.
- Read from and write to relational databases.
- Integrate AI models into structured business workflows.
- Design workflows with logging, retries, monitoring, and recovery.
- Deploy and maintain production-ready n8n systems.
- Document projects professionally.
- Explain technical decisions during interviews.
- Present portfolio projects to employers and clients.

---

# 3. Job-Readiness Levels

## Level 1 — Foundation

The student can follow guided instructions and explain basic automation concepts.

## Level 2 — Applied

The student can build small workflows independently using familiar patterns.

## Level 3 — Professional

The student can design, build, test, document, and debug realistic business automations.

## Level 4 — Job Ready

The student can independently solve open-ended automation problems and defend technical decisions.

# 4. Module Outcomes

## Module 0 — Orientation

After completing this module, students can:

- Install and run n8n locally.
- Navigate the n8n interface.
- Use Git and GitHub for course work.
- Explain the responsibilities of an Automation Engineer.
- Follow the bootcamp workflow and documentation standards.

### Evidence

- Working local n8n installation
- First exported workflow
- Repository setup
- Initial Git commit

---

## Module 1 — Automation Fundamentals

After completing this module, students can:

- Identify triggers, actions, inputs, outputs, and dependencies.
- Describe the lifecycle of an automation workflow.
- Read and explain basic JSON.
- Trace data through a workflow.
- Build a simple end-to-end automation.

### Evidence

- Basic workflow project
- Data-flow explanation
- Short architecture note

---

## Module 2 — Core n8n

After completing this module, students can:

- Build workflows using branching, merging, loops, and conditions.
- Use expressions to reference workflow data.
- Organize workflows for readability.
- Handle common execution failures.
- Refactor a workflow into clearer logical sections.

### Evidence

- Branching workflow
- Loop-based workflow
- Debugging exercise
- Workflow refactor

---

## Module 3 — Data Transformation

After completing this module, students can:

- Clean and normalize structured data.
- Transform arrays and objects.
- Validate required values.
- Convert strings, numbers, and dates.
- Prepare data for APIs and databases.

## Module 4 — APIs

After completing this module, students can:

- Document an external integration.

### Evidence

- Multi-endpoint API integration
- Authentication setup
- Paginated data retrieval
- API error-handling workflow

## Module 5 — Webhooks and Event-Driven Automation

After completing this module, students can:

- Build webhook endpoints.
- Validate incoming payloads.

### Evidence

- Working webhook workflow

---

# 7. Completion Standard

A student is job ready when they can independently:

1. Understand an unfamiliar business process.
2. Ask useful clarification questions.
3. Propose a workflow architecture.
4. Build and test the workflow.
5. Handle common failures.
6. Document the solution.
7. Explain business value.
8. Defend technical decisions.
9. Demonstrate the project professionally.

Course completion alone does not establish job readiness.

Job readiness must be demonstrated through evidence.
